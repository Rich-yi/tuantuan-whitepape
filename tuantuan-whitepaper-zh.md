## 一、项目概述

团团是一套完全自主研发的 AI 加密货币量化交易系统，融合实时新闻情绪分析、多策略组合路由和自进化反馈闭环，已发展为稳定运行的多账号实盘交易平台。

系统采用 Go + Python 双语言微服务架构，当前在 Gate.io 交易所运行 3 个独立账号并行交易，计划扩展至多交易所。

**核心指标一览：**

| 指标 | 数值 |
|------|------|
| 系统架构 | 9 个微服务，Go + Python 双语言 |
| 实盘账号 | 3 个（Gate.io，独立风控隔离） |
| 累计交易 | 3,533 笔，覆盖 52 个交易对 |
| 数据源 | 6+ 实时数据流（新闻、链上、情绪、资金费率） |
| 信号过滤率 | ~88%（仅执行高置信度信号） |
| 风控覆盖 | 三层防护（策略层 + 执行层 + 系统层） |

## 二、市场机会

全球加密货币市场日交易量超 1,000 亿美元。量化交易公司通过以下优势捕获超额收益：

- **信息不对称** — AI 可在数秒内解析数千条新闻和链上信号
- **执行速度** — 亚秒级订单管理 vs 人工交易
- **情绪纪律** — 系统化规则消除恐惧/贪婪驱动的决策
- **全天候运行** — 加密市场永不休市；人需要睡觉，机器不需要

团团的差异化在于其**新闻驱动智能层** — 大多数量化系统仅依赖技术指标，团团则融合 LLM 情绪分析与链上数据，在市场动向被定价之前捕获信号。

## 四、三大核心支柱

### 4.1 新闻驱动智能

不同于仅依赖价格和成交量的传统量化系统，团团通过多源情绪分析管道实时感知市场叙事：

**数据管道：**

```
  CoinTelegraph ───┐
  Decrypt ──────────┤
  CoinDesk ─────────┼──▶ 去重 ──▶ LLM 评分 ──▶ 因子聚合 ──▶ 情绪信号
  BitcoinMagazine ──┤        （极性、影响力、       （时间衰减、
  CoinAnk 链上数据 ──┤          时效性）             加权合并）
  恐惧贪婪指数 ──────┤
  资金费率 ──────────┘
```

- **60 秒刷新周期** — 每分钟完成一轮完整的采集-评分-发布流程
- **多维度评分** — 情绪极性、市场影响力、时间相关性
- **链上数据增强** — CoinAnk 清算量、持仓量、资金费率
- **智能去重** — 防止同一新闻事件重复膨胀情绪分数

### 4.2 多策略自适应路由

三个互补策略通过智能路由层协同决策：

| 策略 | 核心逻辑 | 最佳行情 |
|------|---------|---------|
| **Sentiment Momentum** | 情绪因子驱动的趋势跟随 | 新闻驱动的单边行情 |
| **Mean Reversion** | 价格偏离均值的回归交易 | 区间震荡行情 |
| **Breakout** | 波动率收缩后的突破捕捉 | 盘整后的启动行情 |

**路由器核心能力：**

- **实时行情 Regime 检测** — ADX + 布林带带宽 + ATR 实时分类四种市场状态（趋势/震荡/波动/突破）
- **自适应权重分配** — 70% 基于 Regime 的基础权重 + 30% 基于滚动 Sharpe 的表现权重
- **冲突消解** — 子策略信号冲突时采用加权投票机制
- **智能过滤** — 当 Regime = 震荡且 ADX < 20 时自动暂停开仓，避免手续费损耗

### 4.3 自进化引擎

系统持续从自身交易结果中学习：

- **多维度 PnL 归因** — 按策略、币种、退出原因、持仓时长分解盈亏
- **自动日/周复盘** — 定时生成进化报告，通过钉钉实时推送
- **参数自适应优化** — 基于 EMA 平滑的权重迁移
- **策略淘汰机制** — 表现不佳的策略自动降权至最低阈值（5%）

## 七、优化效果：前后对比

当基于 3,000+ 笔交易的数据驱动分析揭示了结构性低效后，系统进行了系统性重构：

### 7.1 关键参数调整

| 参数 | 优化前 | 优化后 | 调整理由 |
|------|--------|--------|---------|
| 置信度阈值 | 40% | **60%** | 过滤无法覆盖手续费的弱信号 |
| 信号冷却时间 | 15 分钟 | **30 分钟** | 防止同币种频繁翻转 |
| 路由器最低信号强度 | 0.15 | **0.35** | 在源头拦截被稀释的合并信号 |
| 仓位比例 | 余额的 1% | **余额的 5%** | 让单笔交易利润空间足以覆盖手续费 |
| 震荡市过滤 | 无 | **ADX < 20 跳过** | 避免横盘行情中的手续费损耗 |
| 策略配置 | 激进（开发版） | **保守（基准版）** | 信号强度 0.6，确认蜡烛 2 根 |

### 7.2 可量化的结果

| 指标 | 优化前 | 优化后 | 变化 |
|------|--------|--------|------|
| **日均交易** | ~150 笔 | ~99 笔 | **-34%** |
| **胜率** | 12.2% | **41.9%** | **+243%** |
| **手续费效率** | 手续费经常 > 利润 | 比率持续改善 | 结构性修复 |

### 7.3 优化方法论

优化由**基于 SQL 的取证分析**驱动，覆盖全部 3,533 笔历史交易：

1. **手续费结构分析** — 发现 BTC 单合约交易手续费 0.007 USDT，但平均利润仅 0.005 USDT（结构性亏损）
2. **持仓时长归因** — 发现 765 笔 "manual" 退出中 315 笔入场价=出场价（立即翻转，纯手续费损失）
3. **PnL 计算审计** — 发现并修复了一个严重 Bug：Gate.io 的累计仓位 PnL 被错误记录为单笔交易 PnL，涉及 3 条独立的数据库写入路径
4. **Regime 相关性** — 将亏损映射到震荡市周期（ADX < 20），由此引入自动暂停机制
5. **策略胜率分解** — `news` 策略 21.1% 胜率 vs `default` 12.4%，推动权重重新分配

## 八、关键技术突破

### 8.1 精确的合约乘数处理

**挑战：** Gate.io 不同币种的合约乘数（quanto_multiplier）差异巨大（BTC = 0.0001，ETH = 0.01，XRP = 10.0），处理不当导致 BTC 交易的 PnL 计算膨胀高达 10,000 倍。

**解决方案：** 在 PositionManager 中实现 `ContractMultiplierResolver`，当 WebSocket 检测到新仓位时主动查询交易所 API 获取正确乘数。审计并修正了所有三条数据库写入路径，改为从入场/出场价格计算 PnL，不再依赖交易所的累计仓位 PnL 字段。

### 8.2 实时仓位同步

**挑战：** REST 轮询导致仓位状态最多延迟 60 秒，在多账号环境中造成幽灵仓位和遗漏平仓事件。

**解决方案：** 升级为 WebSocket 实时仓位推送，自动保留跨更新的乘数缓存。REST 对账保留为安全网，每账号交错执行（间隔 100 秒）避免 API 限流。

### 8.3 PnL 计算全路径审计

**挑战：** 三条独立的代码路径向数据库写入交易记录，全部错误使用了 Gate.io 的累计 `realised_pnl` 字段（包含资金费和部分平仓历史），而非计算单笔交易的 PnL。

**解决方案：** 系统性审计并修复了所有三条路径：

| 写入路径 | 触发场景 | 退出原因 | 修复方式 |
|---------|---------|---------|---------|
| `publishCompletedTrade` | 系统主动平仓 | 止损、止盈、移动止盈、超时、信号平仓 | 从入场/出场价 × 乘数计算 |
| `publishCompletedTradeFromPosition` | 仓位同步时消失 | 外部平仓 | 从入场/出场价 × 乘数计算 |
| `saveCompletedTrade` | PositionManager 持久化 | 手动平仓 | 从入场/出场价 × 乘数计算 |
